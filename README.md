# 智慧路灯管理平台

基于 Spring Boot 的智慧路灯后端服务，提供设备管理、光照监测、自动/手动控制、告警管理和实时推送功能。

## 通信通道说明

| 通道 | 协议        | 方向      | 触发方   | 用途                    |
|----|-----------|---------|-------|-----------------------|
| ①  | HTTP      | 前端 → 后端 | 前端    | 用户操作：CRUD、查询、手动控制、登录等 |
| ②  | WebSocket | 后端 → 前端 | 硬件/后端 | 实时推送：光照数据、状态变更、告警通知   |
| ③  | JDBC      | 后端 → DB | 后端    | 数据持久化                 |
| ④  | HTTP      | 硬件 → 后端 | 硬件    | 光照上报、心跳上报、开关状态回传、告警触发 |

### 各场景 WebSocket 推送主题汇总

| 场景 | 触发方 | WS 主题 | 消息类型 |
|------|--------|---------|---------|
| 光照数据上报 | 硬件 | `/topic/light-readings` | `LIGHT_REPORTED` |
| 首次光照恢复上线 | 硬件(光照) | `/topic/device-online` | `DEVICE_ONLINE_STATUS_CHANGED` |
| 自动开关灯（阈值触发） | 后端 | `/topic/device-status` | `DEVICE_STATUS_CHANGED` |
| 手动开关灯（前端操作） | 前端 | `/topic/device-status` | `DEVICE_STATUS_CHANGED` |
| 硬件状态回传 | 硬件 | `/topic/device-status` | `DEVICE_STATUS_CHANGED` |
| 心跳超时离线 | 定时任务 | `/topic/device-online` | `DEVICE_ONLINE_STATUS_CHANGED` |
| 独立心跳恢复上线 | 硬件 | `/topic/device-online` | `DEVICE_ONLINE_STATUS_CHANGED` |
| 新告警（离线/执行失败） | 硬件/定时任务 | `/topic/alarms` | `ALARM_CREATED` |

> **后端→硬件指令**：光照上报和心跳上报的 HTTP 响应中统一携带 `{"command": "xxx"}` 字段，硬件根据 command 执行开关。手动控制接口已预留，硬件通知通道待 MQTT 接入。
> **硬件执行失败**：硬件直接调 `POST /alarm-logs` 创建告警，无需走 `status-callback`。
